# Contributing to BrowseFreely

First off, thank you for considering contributing to BrowseFreely! It's people like you that make open-source software such a great community.

### 1. Reporting Bugs
If you find a bug, please create an issue on GitHub with as much detail as possible:
- A clear and descriptive title
- Steps to reproduce the issue
- Expected behavior vs actual behavior
- Relevant logs or screenshots

### 2. Suggesting Enhancements
Have an idea for a new feature? We'd love to hear it! Open an issue using the "Enhancement" label and describe your idea. Explain *why* this feature would be useful to most users.

### 3. Submitting Pull Requests
1. **Fork** the repository.
2. **Clone** your fork locally: `git clone https://github.com/yourusername/browsefreely.git`
3. **Create a branch** for your feature or bugfix: `git checkout -b feature/my-awesome-feature`
4. **Make your changes**. Ensure you write clean, documented TypeScript code.
5. **Test your changes**: Run `bun run dev` and ensure everything works as expected without breaking existing functionality.
6. **Commit** your changes using a descriptive commit message: `git commit -m "feat: add awesome new feature"`
7. **Push** to your branch: `git push origin feature/my-awesome-feature`
8. **Open a Pull Request** against the `main` branch of the original repository.

## üìù Code Style & Guidelines

- **Language**: We use TypeScript in strict mode. Try to avoid `any` where possible.
- **Formatting**: Keep the code clean. We use standard JSX conventions for components.
- **Dependencies**: Since this is meant to be a lightweight tool, please discuss before adding heavy npm packages to the project.
